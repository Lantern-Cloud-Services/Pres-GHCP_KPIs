# Pres-GHCP_KPIs

This repository contains a presentation deck focused on measuring the real impact of GitHub Copilot on software delivery and developer effectiveness. Rather than relying on vanity metrics such as lines of code or team velocity, the presentation promotes a more meaningful approach to KPIs that reflects outcomes, delivery health, and engineering performance.

## What the presentation covers

- Why traditional engineering KPIs can be misleading
- How to evaluate GitHub Copilot using practical, outcome-based metrics
- Four KPI lenses for measuring value: customer usage, pipeline throughput, live-site health, and developer effectiveness
- How dashboards and SPACE-aligned thinking can support better decision-making and coaching
